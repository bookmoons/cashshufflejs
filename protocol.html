Protocol¶

protocol.Phase()¶

Phase enumeration.

Arguments:
  • NONE – Before protocol start.
  • ANNOUNCEMENT – Phase 1. Participants announce themselves.
  • SHUFFLE – Phase 2. Blind shuffling of output addresses.
  • BROADCAST – Phase 3. Verify output addresses.
  • EQUIVOCATION_CHECK – Phase 4. Verify no equivocation.
  • SIGNING – Phase 5a. Sign transaction.
  • VERIFICATION_AND_SUBMISSION – Phase 5b. Submit transaction to network.
  • BLAME – Out of band phase 6. Identify protocol violator.
protocol.Reason()¶

Blame reason enumeration.

Arguments:
  • INSUFFICIENTFUNDS – Insufficient funds.
  • DOUBLESPEND – Double spend.
  • EQUIVOCATIONFAILURE – Equivocation failure.
  • SHUFFLEFAILURE – Shuffle failure.
  • SHUFFLEANDEQUIVOCATIONFAILURE – Shuffle and equivocation failure.
  • INVALIDSIGNATURE – Invalid signature.
  • MISSINGOUTPUT – Missing output.
  • LIAR – Liar.
  • INVALIDFORMAT – Invalid format.
protocol.Invalid()¶

Invalid message.

Provides access to an invalid message as a byte string.

Arguments:
  • invalid (bytes) – Bytes of message.
protocol.Address()¶

Bitcoin Cash address.

Arguments:
  • address (string) – Bitcoin Cash address as string.

Description key.

Arguments:
  • key (string) – Decryption key as string.

protocol.EncryptionKey()¶

Encryption key.

Arguments:
  • key (string) – Encryption key as string.
protocol.Hash()¶

Message hash.

Enables validating message integrity.

Arguments:
  • hash (bytes) – Hash as byte string.
protocol.Registration()¶

Registration request.

Arguments:

protocol.Signature()¶

Digital signature.

Enables verifying message authenticity.

Arguments:
  • signature (bytes) – Signature as byte string.
protocol.Transaction()¶

Bitcoin Cash transaction.

Arguments:
  • transaction (bytes) – Bitcoin Cash transaction as byte string.
protocol.VerificationKey()¶

Verification key.

Public key of a participant. Used to identify sender and recipient in messages.

Arguments:
  • key (string) – Verification key as string.

  • str (string) –

protocol.Packet()¶

Packaged message.

Message with envelope and packaging details.

Arguments:
  • session (bytes) – Session number.
  • number (uint32) –
  • from_key (protocol.VerificationKey) – Sender public key.
  • to_key (protocol.VerificationKey) – Recipient public key.
  • phase (protocol.Phase) – Protocol phase.
  • message (protocol.Message) – Core message.
  • registration (protocol.Registration) – Registration request.
protocol.Signed()¶

Signed packaged message.

Packaged message with signature.

Arguments:
  • packet (protocol.Packet) – Packaged message.
  • signature (protocol.Signature) – Packet signature.
protocol.Packets()¶

List of signed packets.

Arguments:
  • packet (Array.<protocol.Signed>) – Signed packets.
protocol.Blame()¶

Blame details.

Arguments:

  • key (protocol.DecryptionKey) – Decryption key.
  • invalid (protocol.Invalid) – Invalid message.
  • packets (protocol.Packets) – List of signed packets.
